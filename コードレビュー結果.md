### 不具合・バグの指摘と修正提案

#### 1. CC0（Bank MSB）単独変更でプリセットが再構築されない

`renderToWav` で `hasBankSelectInBlock` を設定するが、`programChange` は PC イベントがある場合のみ呼ばれる。

**修正提案:** `hasBankSelectInBlock` が true のときも `programChange(ch, m_channels[ch].program, useBank)` を呼ぶ。

> **✅ 対応済み** `sf_synth.cpp:929-933` — `hasBankSelectInBlock && !needProgramChange` の場合に `programChange()` を呼ぶ分岐を追加。

#### 2. CC1（Modulation）がレンダリングに反映されない

MIDI パースでは CC1 を記録（`midi_file.cpp` L177）するが、`renderToWav` のブロックループに `expr.modulation` 適用がない。`processVoice` は `m_channels[ch].modulation` を参照するため、ビブラートが常に 0。

**修正提案:** volume/expression と同様に `expr.modulation[ch]` をループで `controlChange(ch, 1, v)` する。

> **✅ 対応済み** `sf_synth.cpp:955-957` — `expr.modulation[ch]` をループで `controlChange(ch, 1, v)` に渡すコードを追加。

#### 3. CC65（Portamento On）が `controlChange` で無視される

`renderToWav` は `expr.portamentoOn` を `controlChange(ch, 65, v)` に渡すが、内部で破棄される。`noteOn` のポルタメント処理が機能しない。

**修正提案:** `case 65: ch.portamentoOn = value; break;`

> **✅ 対応済み** `sf_synth.cpp:500` — `case 65: break;` を `case 65: ch.portamentoOn = value; break;` に変更。

#### 4. `--pitch` 使用時の noteOff 不一致

noteOn では pitchShift を適用するが noteOff では適用していない。

**修正提案:** noteOff でも同じ `pitchShift` を適用する。

#### 5. `tickForTime` 逆変換が不正確

秒→tick の線形補間がなく、テンポ変更後の CC/PC/SysEx タイミングがずれる。

**修正提案:** `tickToSeconds` の逆関数を実装し、区間ごとに BPM で補間する。

#### 6. SC-8850 モデル検出のオフセット誤り

Identity Reply でモデル ID のオフセットが誤っている（`midi_file.cpp` L227–231）。

**修正提案:** `addr1==0 && addr2==0 && addr3==0` かつ `data[0..2] == {0x54,0x00,0x14}` で判定。

#### 7. SysEx 後の Running Status 汚染

SysEx（0xF0）処理後、`runningStatus = 0xF0` のまま残る。MIDI 仕様では SysEx は Running Status をクリアする。

**修正提案:** SysEx 処理後に `runningStatus = 0`。

#### 8. RPN 1（Fine Tune）の Data Entry 解釈が誤り

RPN 1 は 14bit（CC6+CC38、中心 8192）。MIDI パーサが RPN を一切記録していない（CC98–101, CC6/38 未対応）。

#### 9. トラック終端で noteOn 残存

`extractNotes` はトラック末尾で鳴り続けるノートをフラッシュしない。

#### 10. 同一チャンネル・同一ノートのポリフォニー追跡不可

`noteOn[channel][note]` は 1 エントリのみ。

#### 11. `renderToWavPerChannel` が CC/エフェクト未適用

チャンネル分割出力は program/bank 初期化のみ。

#### 12. SF2 Modulator リスト未処理

`presetBags`/`instBags` の `modNdx` を無視。

- Variable-length delta、Running Status、NoteOn vel=0 as NoteOff

## レビュー #2 — 2026-06-21

**対象:** WAV 再現忠実性（`midi_file.cpp` → `sf_synth.cpp` → `wav_writer.cpp`）  
**方針:** 「パースはできているがレンダリングで失われる」断絶と、アーキテクチャ上のタイミング誤差を重点調査

---

### 不具合・バグの指摘と修正提案（WAV reproduction focused）

#### 1. 【致命的】`tickForTime` により CC/表現イベントがブロックに適用されない

```889:902:cpp/src/sf_synth.cpp
            auto tickForTime = [&](double sec) -> int64_t {
                int64_t tick = 0;
                // ...
                for (auto& [t, bpm] : tmap) {
                    double thisTime = midi.tickToSeconds(t);
                    if (thisTime >= sec) break;
                    tick = t;
                }
                return tick;
            };
            blockTickStart = tickForTime(blockSecStart);
            blockTickEnd = tickForTime(blockSecEnd);
```

- テンポ変更点の tick のみ返し、**秒→tick の線形補間がない**
- 単一テンポ MIDI では `blockTickStart == blockTickEnd == 0` が常に成立
- 適用条件 `t >= blockTickStart && t < blockTickEnd`（L912, L928 等）が **常に false** → Volume/Expression/Pan/Sustain/Pitch Bend/PC がレンダ中に無視される
- **修正:** `tickToSeconds` の逆関数（区間 BPM 補間）を実装。`blockTickEnd` はブロック終端 tick（排他的）を正確に計算。`blockTickStart == blockTickEnd` のときは `blockTickEnd + 1` 等で空区間を回避

#### 2. 【致命的】`processVoice` のピッチ進行が `pitchRatio` を使わない

```714:731:cpp/src/sf_synth.cpp
        double effectivePitchRatio = v.basePitchRatio;
        if (v.portamentoProgress < 1.0) {
            // ... 補間 ...
            effectivePitchRatio = v.basePitchRatio + (v.pitchRatio - v.basePitchRatio) * t;
        }
        v.position += effectivePitchRatio * std::pow(2.0, vibratoShift / 12.0);
```

- ポルタメントなし（`portamentoProgress == 1.0`）では `basePitchRatio`（既定 **1.0**）のみ使用
- `startVoice`（L274–286）は `v.pitchRatio` を正しく計算するが **`basePitchRatio` にコピーしない**
- メロディはルートキー付近の固定ピッチで再生され、**音程が MIDI と一致しない**
- ドラムは `pitchRatio ≈ sampleRate/44100 ≈ 1.0` のため「リズムはそれなり、メロディがおかしい」という症状と一致
- **修正:** `startVoice` で `v.basePitchRatio = v.pitchRatio`、または `else` で `effectivePitchRatio = v.pitchRatio`

#### 3. チャンネル初期化が program/bank のみで CC 状態を復元しない

```824:851:cpp/src/sf_synth.cpp
    for (int ch = 0; ch < 16; ch++) {
        // firstNoteTick 以前の PC/Bank のみ探索
        // volume/expression/pan/sustain/pitchBend は未設定
        programChange(ch, m_channels[ch].program, m_channels[ch].bank);
    }
```

- `ChannelState` 既定値（volume=100, expression=127, pan=64）のまま開始
- 最初のノート以前の CC7/CC11/CC10/CC64 等が無視される
- `MidiExpression::getValueAtTick`（`midi_file.h` L125–128）はあるが未使用
- **修正:** 各 ch の `firstNoteTick` で `getValueAtTick` により全 CC を初期化してから `programChange`

#### 4. CC1（Modulation）が `renderToWav` ループに存在しない

- パース: `midi_file.cpp` L177
- 合成: `controlChange` case 1（`sf_synth.cpp` L477）は実装済み、`processVoice`（L581–583）も参照
- レンダループ（L926–957）に **modulation 適用ループがない**（Review #1 の指摘を再確認）

#### 5. CC0 単独変更でプリセット再構築されない（Review #1 継続）

- L909 `hasBankSelectInBlock` を設定するが L920 の `needProgramChange` とは連動しない
- Bank MSB のみ変更時 `buildPresetZones` が呼ばれない

#### 6. noteOn/Off のブロック量子化と noteOff ロジックの非対称

**メインパス（`renderToWav`）:**
- noteOn: ブロック内サンプル位置で発火（L962–963）— ブロック先頭に量子化
- noteOff: `noteEnd <= pos`（L980）— **ブロック開始時のみ判定**、最大 ~23ms（1024/44100）遅延
- `--pitch` 使用時: noteOn は `pitchShift` 適用（L970）、noteOff は未適用（L981）→ 音が切れない

**Per-channel パス（`renderToWavPerChannel`）:**
- noteOff: `noteEnd >= pos && noteEnd < blockEnd`（L1203）— メインパスと**異なる**
- CC/エフェクト/初期表現状態は**一切未適用**（L1177–1209）
- noteOff にも `pitchShift` 未適用（L1204）

#### 7. ピッチベンド・パンはノートオン時に固定

- `startVoice`（L277, L294–295）で `pitchRatio`/`v.pan` を固定設定
- `processVoice` は volume/expression を毎サンプル参照（L690–691）するが、**pan と pitch bend はライブ更新しない**
- CC が正しく適用されても、ノート保持中のベンド/パン変化は再現されない

#### 8. エフェクトが `processBlock` の後に適用される

```988:1049:cpp/src/sf_synth.cpp
        processBlock(bl, br, blockLen);
        // この後に SysEx エフェクトレベル更新 + reverb/chorus/delay 適用
```

- ブロック内 SysEx イベントのレベル変更が**同ブロックの乾音生成後**に反映
- 1 ブロック分のタイミングずれ

#### 9. ピッチベンドは新規ノートにのみ影響

- 既存ボイスの `pitchRatio` は noteOn 時固定
- リアルタイム MIDI 再生器と異なるアーキテクチャ

#### 10. `extractNotes` とレンダリングの構造的ギャップ

| 項目 | extractNotes | renderToWav |
|------|-------------|-------------|
| ノート時刻 | tick ベース | `tickToSeconds` → サンプル（OK） |
| ベロシティ | NoteOn 時点 | そのまま使用（CC7×CC11 未反映） |

- **noteOn:** ブロック内の任意 tick でも **ブロック先頭**で発火 → 最大 ~23ms **早鳴り**
- **noteOff:** `noteEnd <= pos` は **次ブロック開始時**にのみ判定 → 最大 ~23ms **遅延リリース**
- 短いノート（スタッカート、ドラム）は特に「濁る／伸びる／不自然」
- Per-channel パスは noteOff が `noteEnd >= pos && noteEnd < blockEnd`（`1290-1293`）と**非対称**

**修正提案:** ブロック内サンプルオフセットで noteOn/Off。またはイベント駆動レンダラへ統一。

#### 2. 【高】RPN/CC6/CC38/CC98–101 が MIDI からレンダに届かない

### 不具合・バグの指摘と修正提案

#### 1. 【致命的】`sampleType` ループ判定が SF2 仕様と不一致

```216:216:cpp/src/sf_synth.cpp
 z.loop = (s.sampleType & 0x1000) != 0;
```

SF2 仕様では `sampleType`（16bit）の**上位バイト（bits 8–15）がループモード**。

| 典型値 | 意味 | `(x & 0x1000)` |
|--------|------|----------------|
| `0x0101` | mono + loop continuously | **false**（bit 12 未セット） |
| `0x0001` | mono + no loop | false |

GM ピアノの `0x0101` 等は**ループありなのに判定 false** → ほぼ全サステイン楽器でループ無効。

`processVoice` ではループ OFF 時、`sampleEnd` 到達でボイス強制終了:

```653:663:cpp/src/sf_synth.cpp
 if (v.loop && v.loopEnd > v.loopStart) {
 // ... loop ...
 } else if (idx >= (int)v.sampleEnd) {
 v.active = false;
 break;
 }
```

**修正提案:**

```cpp
int loopMode = (s.sampleType >> 8) & 0xFF;
z.loop = (loopMode == 1 || loopMode == 3); // 1=continuous, 3=loop until release
```

FluidSynth 等と同様に mode 3 は「全サンプル再生後ループ」も実装するとより忠実。

#### 2. 【中】`fineTune` の rootKey への二重・誤適用

```128:128:cpp/src/sf_synth.cpp
 case 24: def.fineTune = a / 100.0; break; // セント→半音
```

```209:210:cpp/src/sf_synth.cpp
 z.rootKey += z.fineTune / 100.0; // fineTune は既に半音単位なのに再度 /100
```

**修正:** `z.rootKey += z.fineTune;`

#### 3. 【低】Program/Bank 誤マッピング（PC 0→105）は可能性低

| 調査項目 | 結果 |

| MIDI PC パース | 0–127 そのまま — off-by-one なし |
| Ch10 強制 bank=128 | `ch==9` のみ — メロディ Ch には非適用 |
