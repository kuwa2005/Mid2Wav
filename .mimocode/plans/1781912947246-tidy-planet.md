# Mid2Wav チューニングフェーズ — 「2026年版MIDI音源」コンセプト

## 現状の音質特性と課題

| 部分 | 現状 | 課題 |
|------|------|------|
| サンプル補間 | Lanczos-2 (4-tap) | 高音域でエイリアシング残存 |
| エンベロープ | 線形ADS | SF2仕様の指数カーブに未準拠、音が機械的 |
| フィルタ | Biquad LP のみ | SF2 Gen39-42(モジュレーションエンベロープ→フィルタ)未実装 |
| リバーブ | モノFDN | ステレオ画像破壊、プリディレイなし |
| コーラス | モノsum+整数遅延 | ステレオ定位崩壊、ジッパー雑音 |
| ディレイ | 整数遅延 | クリックノイズ、クロスフィードバックなし |
| 出力 | 44100Hz/16bit | ダイナミックレンジ不足、高域特性限界 |
| ボイスミックス | 加算のみ | クリッピング、ボイススティールなし |

## 改善アイデア（優先度順）

### Tier 1: 音質に直結する改善（高効果・中難度）

#### 1. エンベロープの指数カーブ化
- **現状**: 線形ADSR（`attackRate = 1.0 / (attackTime * sampleRate)`）
- **改善**: SF2仕様準拠の指数減衰。`envLevel *= decayMultiplier` 形式
- **効果**: アタック/リリースの減衰曲線が自然になり、アコースティック楽器の響きが大幅改善
- **根拠**: 線形エンベロープは「古き良き時代の音源」の最大の原因

#### 2. サンプル補間のLanczos-3/4化
- **現状**: Lanczos-2 (4-tap)
- **改善**: Lanczos-3 (6-tap) またはwindowed-sinc 8-tap
- **効果**: 高音域のエイリアシング低減。ピアノ高音部やフルートの倍音がクリアに
- **難度**: 低（カーネル定数の変更のみ）

#### 3. 24bit WAV出力対応
- **現状**: 16bit固定、ditheringなし
- **改善**: `--bitdepth 16|24|32float` オプション追加。24bit時はTPDF dithering
- **効果**: ダイナミックレンジ14bit→23bit。 pianissimo のディテールが完全保持

#### 4. 出力サンプルレートの可変
- **現状**: 44100Hz固定
- **改善**: `--samplerate 44100|48000|88200|96000` オプション
- **効果**: 96kHz出力で超音波域の倍音を保持。DAWとの統合性向上

### Tier 2: SF2仕様の完全準拠（中効果・中難度）

#### 5. モジュレーションエンベロープ実装
- **現状**: SF2 Gen39-42 未実装
- **改善**: Filter Mod Env (Gen39), Volume Mod Env (Gen39), Delay (Gen25) 実装
- **効果**: エレピやストリングスの減衰フィルタ変化が再現される

#### 6. ベロシティ→フィルタ周波数マッピング
- **現状**: Gen44 (velocity-to-filter) 未実装
- **改善**: `filterFc = baseFc + velocity * scale`
- **効果**: 強いアタック時に明るい音、弱いアタック時に暗い音 — 表現力向上

#### 7. SF2 Modulator対応
- **現状**: Gen46-59 の modulator 未処理
- **改善**: 基本的なunipolar/bipolar modulator実装
- **効果**: Velocity-to-attack, Pressure-to-filter 等のSF2独自表現が可能に

### Tier 3: エフェクト品質向上（中効果・中難度）

#### 8. ステレオFDNリバーブ
- **現状**: モノ入力-sum → FDN
- **改善**: 入力をステレオのまま処理。combフィルタにL/R個別のディレイタイム設定
- **効果**: リバーブ残響がステレオのまま残り、音場が広がる

#### 9. ステレオコーラス + 分数遅延補間
- **現状**: モノsum、整数遅延
- **改善**: L/R独立LFO（位相差あり）、Lanczos補間された分数遅延
- **効果**: ジッパー雑音消滅、ステレオ定位の安定化

#### 10. リバーブにプリディレイ + モジュレーション
- **現状**: なし
- **改善**: combフィルタ入力前に10-50msのディレイ、comb遅延ラインに0.1-1HzのLFO
- **効果**: 残響の「固さ」が解消され、リアルな空間感が向上

### Tier 4: 高度なDSP（高効果・高難度）

#### 11. 2x オーバーサンプリング（ボイス処理時）
- **現状**: 出力レートでそのまま処理
- **改善**: processVoice内部で2倍にサンプリング→LPF→出力レートに戻す
- **効果**: フィルタエイリアシング消滅、高音域の倍音保持

#### 12. ボイスレベリング / マスターリミッター
- **現状**: 加算のみ、後で正規化
- **改善**: ブロック単位のRMS計測→ブロック間のゲインスムージング→tan/clamp
- **効果**: ダイナミクスが自然になり、正規化による音圧崩壊を防止

#### 13. ループクロスフェード
- **現状**: ループ境界で直接ラップ
- **改善**: ループ点前後128-512サンプルでクロスフェード
- **効果**: ループノイズ（-click, ビーン）が完全消滅

## 実装計画 — Phase A: 基本音質改善（一括実装）

### A1. エンベロープの指数カーブ化
**対象ファイル**: `cpp/src/sf_synth.cpp`
**変更内容**:
- attack: `envLevel += attackRate` → `envLevel = 1.0 - (1.0 - envLevel) * attackCoeff`（指数上昇）
- decay: `envLevel -= decayRate` → `envLevel *= decayCoeff`（指数減衰、SF2 spec準拠）
- release: 同様に指数減衰
- `attackRate/decayRate/releaseRate` を `attackCoeff/decayCoeff/releaseCoeff` に変更

### A2. サンプル補間をLanczos-3に改善
**対象ファイル**: `cpp/src/sf_synth.cpp`
**変更内容**:
- `getSample()` 関数をLanczos-3 (6-tap) に書き換え
- カーネル: `radius=3`, `sin(πt)sin(πt/3) / (πt·πt/3)`
- 6サンプル参照に拡大

### A3. 24bit WAV出力対応
**対象ファイル**: `cpp/src/wav_writer.cpp`, `cpp/include/wav_writer.h`, `cpp/src/converter.cpp`, `cpp/src/main.cpp`
**変更内容**:
- `--bitdepth 16|24|32float` オプション追加
- WAVヘッダの `bitsPerSample` を動的に設定
- 24bit時: float→int24変換 + TPDF dithering
- 32bit float時: clampのみ

### A4. 出力サンプルレート可変化
**対象ファイル**: `cpp/src/converter.cpp`, `cpp/src/main.cpp`
**変更内容**:
- `--samplerate 44100|48000|88200|96000` オプション追加
- `synth.init(sf2, sampleRate)` に指定値を渡す
- WAVヘッダの `sampleRate` を動的に設定
- SF2サンプルの補間パスが自動的に適応

### 検証方法
1. `./Mid2Wav --no-normalize --bitdepth 24 --samplerate 96000 -i test.mid -o out/` で24bit/96kHz出力確認
2. WAVヘッダのbyte rate, bits per sampleをhexdumpで検証
3. RMS/Peak値が従来と大きく乖離しないことを確認
4. エンベロープが自然な減衰曲線を描くことを数値的・聴覚的に確認
