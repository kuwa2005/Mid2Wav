# コードレビュー結果の妥当性検証 — 修正計画

## 検証結果サマリー

レビュー#1・#2の全指摘事項を現在のコードと照合。**未修正の重大バグが7件**見つかりました。

## 未修正バグ一覧（優先度順）

### 【致命的】basePitchRatioが常に1.0（レビュー#2-2）
- **コード**: `sf_synth.cpp:714` → `effectivePitchRatio = v.basePitchRatio` (デフォルト1.0)
- **原因**: `startVoice`で`v.basePitchRatio = v.pitchRatio`が呼ばれない。ポルタメント時のみ`v.basePitchRatio = prevPitchRatio`(L439)
- **影響**: 全ノートがルートキー(C4)付近の固定ピッチで再生される。メロディの音程が完全にずれる
- **修正**: `startVoice`の末尾に`v.basePitchRatio = v.pitchRatio`を追加

### 【致命的】CC65(Portamento On)がcontrolChangeで無視される（レビュー#1-3）
- **コード**: `sf_synth.cpp:500` → `case 65: break;`
- **影響**: ポルタメントON/OFFが永続的に無効。`ch.portamentoOn`が更新されない
- **修正**: `case 65: ch.portamentoOn = value; break;`に変更

### 【致命的】CC1(Modulation)がrenderToWavのCC適用ループに存在しない（レビュー#1-2、#2-4）
- **コード**: `sf_synth.cpp:926-957`のCC適用ループにmodulationの適用がない
- **影響**: CC1のビブラート深度がレンダ中に反映されない。always 0
- **修正**: `expr.modulation[ch]`の適用ループを追加

### 【重要】CC0(Bank MSB)単独変更でプリセット再構築されない（レビュー#1-1、#2-5）
- **コード**: `sf_synth.cpp:912`で`hasBankSelectInBlock=true`だが、L920の`needProgramChange`とは連動しない
- **影響**: Bank MSBのみ変更時、音色が変わらない
- **修正**: `hasBankSelectInBlock && !needProgramChange`のときもprogramChangeを呼ぶ

### 【重要】voice.ageがインクリメントされない（レビュー#2-11）
- **コード**: `v.age`は宣言のみで`age++`がどこにもない
- **影響**: ボイススティールが常に同じボイスを選択（最古判定が機能しない）
- **修正**: `processBlock`または`processVoice`で`v.age++`を追加

### 【重要】tickForTime逆変換が不正確（レビュー#2-1、#1-5）
- **コード**: `sf_synth.cpp:878-884` → テンポ変更tickの返却のみ、秒→tickの線形補間なし
- **影響**: 単一テンポMIDIで`blockTickStart == blockTickEnd`が常に成立→CC/PC/SysExが全て無視される
- **修正**: `tickToSeconds`の逆関数を実装。区間BPM補間で正確なtickを計算

### 【重要】processVoiceでピッチベンド/パンがライブ更新されない（レビュー#2-7、#2-9）
- **コード**: `processVoice`はvolume/expressionを毎サンプル参照するが、panとpitchRatioはstartVoiceで固定
- **影響**: ノート中のピッチベンド/パン変化が再現されない
- **修正**: processVoice冒頭でpanとpitchRatioを再計算
